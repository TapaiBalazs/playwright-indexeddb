How this form cache works

This showcase demonstrates saving a form into IndexedDB as a single key-value entry while the user is typing.

Use this page in the following way:

  1. Enter values into the form fields. Required fields must be filled before the form can be submitted.
  2. Pause for about one second after typing. The current form state is then written to IndexedDB automatically.
  3. Reload the page to confirm that previously saved values are restored from IndexedDB and placed back into the form.
  4. Complete the required fields and submit the form. Submission clears the saved IndexedDB entry and resets the form.

This example is useful when you want to preserve in-progress form data between reloads, route changes, or temporary browser interruptions.
